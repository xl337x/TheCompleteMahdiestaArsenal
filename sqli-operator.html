🎯 Step 1.5: Column Type Detection & Matching

        CRITICAL
      

      

        

          ⚠️ Type Mismatch Problem: If you see errors like:

          • UNION types integer and text cannot be matched

          • invalid input syntax for type integer

          • UNION types character varying and integer cannot be matched


          This means you need to detect column types and use CAST/CONVERT to match them!

          💡 Type-Safe Extraction Strategy:

          When UNION fails due to type mismatch, use Error-Based Extraction instead:

          • PostgreSQL: CAST(version() AS INTEGER) → error contains version!

          • MySQL: EXTRACTVALUE() or UPDATEXML()

          • MSSQL: CONVERT(INT, @@version)

          The error message becomes your data exfiltration channel!
